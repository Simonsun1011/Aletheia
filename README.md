# Aletheia

面向中期美股科技投资的**研究与判断支持系统**。

Aletheia 用于收束市场信息、呈现量化状态、组织正反证据、记录个人判断，并在事后对照结果、检验判断质量。它不是预测器、信号机或自动交易工具——判断权始终由使用者保留。

AI 是可选的辅助层，主要用于摘要、结构化、知识讲解和带来源的外部观点整理；不配置模型服务时，本地数据、判断记录、复盘、术语库和 Obsidian 等核心流程仍可使用。AI **不得以自身名义输出买卖建议、多空结论、目标价或仓位建议**。

核心流程：

> 发生了什么 → 数据如何变化 → 可能影响什么 → 支持与反对证据 → 我如何判断 → 是否行动 → 后来事实证明了什么

---

## 功能

| 模块 | 做什么 |
|------|--------|
| **信息流** | 收束市场信息并生成盘后批量卡片，呈现事实摘要、来源、相关标的与术语；中文译文可缓存复用 |
| **操作台** | 按持仓或标的查看宏观、基本面、技术面与叙事扫描，分开展示多方观点、中立事实与空方观点 |
| **标的快照** | 展示 ATR、RSI、布林带等经典公开指标，不生成综合评分或黑箱信号 |
| **术语库** | 管理中英别名、正文自动高亮、教科书级解释、出现情境与用户笔记，并支持导出到 Obsidian |
| **判断与复盘** | 判断记录采用 append-only：原文不可覆盖或删除，只能追加修正、撤回和复盘；支持期限、方向、置信度与事后校准 |
| **设置** | 管理术语学习状态、Obsidian 导出目录、模型与本地运行配置等 |

个人判断、随感、成交、买入方案和本地数据库不会提交到本仓库。Aletheia 默认本机优先运行，密钥只从本机环境变量读取。

## 用法

**环境**：Python 3.10+、Node.js 18+。

```bash
git clone https://github.com/Simonsun1011/Aletheia.git
cd Aletheia

python3 -m venv .venv
.venv/bin/pip install -r backend/requirements.txt

cp .env.example .env
# 用任意文本编辑器打开 .env，按下面「配置说明」填写

cd frontend && npm install && cd ..

./scripts/dev.sh
# 端口被占用时：./scripts/dev.sh --kill-ports
```

浏览器打开：http://127.0.0.1:3000

### 配置说明（`.env`）

`.env` 是本机私密配置文件，**不要发给别人，也不要提交到 GitHub**。仓库里只有模板 `.env.example`：先复制成 `.env`，再修改其中的值。

用记事本、VS Code、Cursor 等打开 `.env` 即可。以 `#` 开头的是注释，`名称=值` 表示一项配置。多数配置已有默认值；只有需要启用 AI 摘要、翻译或叙事扫描时，才需要配置相应的模型服务。Aletheia 不绑定特定供应商，可通过 LiteLLM 或 OpenAI 兼容端点接入不同模型。

| 配置项 | 要不要填 | 含义 |
|--------|----------|------|
| `OPENAI_API_KEY` | 按需 | OpenAI 兼容调用使用的密钥。变量名沿用兼容接口约定，不代表产品绑定 OpenAI；可填写所选兼容服务商提供的密钥。未配置时，仅相关 AI 功能不可用。 |
| `OPENAI_API_BASE` | 按需 | OpenAI 兼容端点地址。使用兼容服务时填写服务商提供的地址，例如 `https://example.com/v1`；由 LiteLLM 直接管理的其他供应商可按其配置方式接入。 |
| `MODEL_SUMMARY` | 可选 | 摘要与结构化任务使用的模型名；模板中已有默认值。 |
| `MODEL_SEARCH` | 可选 | 操作台叙事扫描使用的联网搜索模型。未配置时，该功能禁用，其他功能仍可使用。 |
| `ANTHROPIC_API_KEY` | 按需 | 使用 Anthropic 模型时配置；其他供应商使用各自对应的环境变量或兼容端点。 |
| `MONTHLY_LLM_BUDGET_USD` | 可选 | 每月 AI 费用预算。超限后部分批处理会跳过 AI；交互式调用可能继续执行并显示警告。 |
| `OBSIDIAN_EXPORT_DIR` | 可选 | Obsidian 笔记目录的绝对路径。未配置时，“加入知识笔记”功能不可用。 |
| `NEXT_PUBLIC_API_BASE` | 一般不用 | 前端访问后端的地址，默认是 `http://127.0.0.1:8000/api`；只有修改后端端口时才需要同步调整。 |

数据库、日志、CORS 和其他路径配置通常可以保持模板默认值。密钥只应保存在你自己的 `.env` 中，不要写入代码、数据库或提交记录。

## 设计原则

- 判断权始终由使用者保留，AI 不代替投资决策。
- 事实、指标、AI 摘要、外部观点和个人判断在数据结构与界面中保持分离。
- 判断记录采用 append-only，所有修正、撤回和复盘均保留时间与历史链条。
- 只使用经典公开指标，不制造不可解释的综合评分或黑箱信号。
- 复盘与校准始终显示样本量；样本不足时明确提示统计意义有限。
- 本机优先，密钥只从环境变量读取。
